# Jornadas do Usuário

No design de experiência do usuário (UX), **user journey** (jornada do usuário) e
**user flow** (fluxo do usuário) são ferramentas complementares que olham para a
interação a partir de níveis diferentes: uma com foco na **empatia** e outra com
foco na **funcionalidade**.

## User Journey x User Flow

- **User Journey (empatia):** é a representação visual da história completa da
  interação de um usuário com um produto, serviço ou marca, desde o momento em que
  ele descobre uma necessidade até a fase pós-uso. Olha para o **estado emocional**,
  os pensamentos, as motivações e os pontos de dor.
- **User Flow (funcionalidade):** é um diagrama que ilustra as etapas específicas,
  lógicas e passo a passo que um usuário percorre para concluir **um objetivo
  único** dentro de uma interface. Olha para os detalhes técnicos, a usabilidade, a
  navegação tela a tela e as funcionalidades.

Em resumo, a jornada conta a história inteira do ponto de vista de quem usa,
enquanto o fluxo funciona como um **zoom** em uma etapa específica dessa história.

## Por que analisar a jornada

Mapear a jornada do usuário ajuda a equipe a:

- **encontrar lacunas** no serviço;
- **descobrir insights** que não apareceriam de outra forma;
- atuar como uma **ponte entre os silos** da organização (um *objeto de fronteira*),
  em que cada área enxerga oportunidades diferentes de melhoria;
- **entender o ponto de vista do usuário**;
- combinar-se com outras ferramentas, como as **personas**, para estender o
  entendimento a diferentes perfis de usuário.

## Montando um mapa de user journey

Alguns pontos ajudam a construir um bom mapa de jornada:

- **Escala:** ela pode variar, e entender qual escala analisar é decisivo para gerar
  um insight de qualidade.
- **Fontes:** a análise pode começar com pressuposições, mas o ideal é que evolua
  para dados de **pesquisa**.
- **Real x ideal:** os mapas podem representar tanto a jornada **atual** quanto uma
  jornada **nova**, usada para analisar uma mudança antes de implementá-la.

A jornada pode analisar tanto um produto quanto uma experiência mais ampla.

### Exemplo de jornada

Pensando em um marketplace, a jornada pode ser lida em etapas que afetam os
sentimentos e as decisões do usuário:

1. **Descoberta:** o usuário descobre o mundo das compras online.
2. **Consideração:** conhece o marketplace por meio de um anúncio.
3. **Decisão:** decide fazer a primeira compra na plataforma.
4. **Retenção:** o programa de cashback faz o usuário continuar usando a plataforma.

## User Flow: o detalhamento micro

O user flow atua como um **zoom da jornada**, com foco em uma etapa específica.
Suas características principais são:

- foco em **uma etapa específica** e no mapeamento de objetivos granulares;
- **elicitação das etapas** que o usuário percorre;
- análise dos **pontos de contato críticos**;
- um olhar **técnico e sequencial**, analisando as linhas de conexão entre os passos.

### Exemplo de fluxo

Para um usuário recorrente de um aplicativo de compras, um fluxo possível seria:
**abrir a aplicação → pesquisar o produto desejado → comparar produtos concorrentes
→ finalizar a compra** (ou, em um caminho alternativo, **desistir da compra**).

### Diferenças entre as duas ferramentas

O user flow prioriza a **sequência técnica** dos passos, sem linhas visuais ou
citações, com foco na funcionalidade e na execução das atividades. Já o journey map
é uma **grade mais complexa**, que reúne emoções e pensamentos (muitas vezes em
forma de citações), gráficos e outros elementos visuais.

## Jornada, normas e acessibilidade

A forma como as normas e os padrões de interface são aplicados influencia
diretamente a jornada e tem impacto direto na acessibilidade:

- **Mensagens de erro** devem ser visíveis, ficar próximas ao campo e trazer
  sugestão de correção, ajudando o usuário na etapa atual da jornada.
- **Modais** mudam a jornada de forma abrupta e, por isso, devem ser evitados.
- **Formulários críticos** devem permitir revisar ou reverter o envio, transmitindo
  uma sensação de segurança.

No quesito **navegação e teclado**, vale a máxima "se não funciona no teclado, não
funciona":

- toda funcionalidade deve ser acessível **sem o mouse**;
- o **foco visível** é obrigatório em todos os elementos interativos;
- *dropdowns* e submenus precisam **abrir e fechar pelo teclado**;
- a ordem de foco deve ser **lógica**, de cima para baixo e da esquerda para a
  direita.

Mapear a jornada considerando esses cuidados garante que a experiência seja
inclusiva em cada etapa, e não apenas em telas isoladas.
